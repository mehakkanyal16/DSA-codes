## 📁 Structure

Each problem gets its own folder, named `<problem-number>-<problem-slug>`, containing the solution file and a short auto-generated README with the submission's runtime and memory stats.

```
DSA-codes/
├── 1-two-sum/
│   ├── two-sum.java
│   └── README.md
├── 101-symmetric-tree/
│   ├── symmetric-tree.cpp
│   └── README.md
├── 1036-rotting-oranges/
│   ├── rotting-oranges.java
│   └── README.md
└── ...
```

## 🔍 Finding a problem

Press **`t`** on any GitHub page in this repo to open file search, then type the problem name or number to jump straight to its folder.

## ⚙️ How it works

1. Solve a problem on LeetCode.
2. Submission gets accepted.
3. [LeetSync](https://github.com/apps/leetsync) pushes the solution + a stats README to this repo — automatically.

## 🛠️ Languages

- **Java** — primary language for most solutions
- **C++** — used for a subset of problems
- **SQL** — database/query-based problems

## 📌 Notes

This is a personal practice log, not a curated tutorial repo — solutions reflect whatever passed on LeetCode at the time, not necessarily the most optimal approach.
